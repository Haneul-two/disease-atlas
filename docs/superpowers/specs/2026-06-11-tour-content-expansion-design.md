## 1. 퀵 투어 모드

### 데이터 (정적 TS 파일 — A안)

투어는 자주 바뀌지 않는 큐레이션 콘텐츠이므로 DB가 아닌 코드에 둔다.
DB 스키마 변경·마이그레이션·admin 작업 없음.

`src/lib/tours.ts`:

```ts
type TourStep = {
  diseaseSlug: string;   // DB id 대신 slug로 참조 (시드 간 안정적)
  narrative: string;     // 이 스텝에서 보여줄 해설 (2~3문장, 교육용 어조)
};
type Tour = {
  slug: string;
  title: string;         // 예: "치매로 가는 길"
  description: string;   // 투어 목록에 보일 한 줄 소개
  steps: TourStep[];     // 5~8 스텝
};
```

- `atlas-data.ts`의 노드에 `slug` 필드 추가 (현재는 id·name만) → 렌더 시 slug→노드 매핑.
- 존재하지 않는 slug의 스텝은 건너뛰고 dev 콘솔에 경고.

### 투어 콘텐츠 (노인성 질환 중심 3개)

1. **"치매로 가는 길"** — 고혈압 → 뇌경색 → 혈관성치매 → 알츠하이머 → 정상압수두증
   (혈관 위험인자에서 인지저하까지)
2. **"떨림의 정체"** — 본태성진전 → 파킨슨병 → 이차성파킨슨증 → 뇌졸중후유증
   (비슷해 보이는 떨림·운동장애 구분)
3. **"심장에서 뇌까지"** — 고지혈증 → 고혈압 → 협심증 → 심근경색 → 심부전 → 뇌졸중
   (하나의 혈관 스토리)

### UI / 동작

- 헤더에 **"투어" 버튼** → 투어 목록 카드(제목 + 한 줄 소개).
- 투어 시작 시 **TourCard 패널**(데스크톱: 좌하단, 모바일: 하단 시트):
  `스텝 n/N · 해설 · 이전/다음/종료`.
- 스텝 이동 시 기존 `focusNode()` 재사용 — 카메라 이동 + 노드 선택
  (이웃 강조·실루엣 글로우 그대로 작동, 숨긴 부위 자동 표시).
- 투어 중에는 DetailPanel 대신 TourCard 표시 (해설이 그 역할). 종료 시 일반 모드 복귀.
- 키보드: `←`/`→` 스텝 이동, `Esc` 종료.

### 파일

- 신규: `src/lib/tours.ts`, `src/components/atlas/TourCard.tsx`, `src/components/atlas/TourMenu.tsx`
- 수정: `AtlasFlow.tsx`에 투어 상태(tourSlug + stepIndex) 추가, `atlas-data.ts`에 노드 slug 포함

## 2. 콘텐츠 확충 (46 → ~80개)

- `prisma/seed.ts`에 질병 ~34개 추가, **부위당 ~10개 균형**
  (현재 뇌·신경 치우침 → 가슴·복부·관절·내분비 위주 보강).
- 각 질병: 설명 · 치료법 · 증상 3~4개. 노인성 질환은 합병·연관 관계도 함께 추가.
- 시드는 기존 upsert 방식 유지(비파괴). **배포 후 프로덕션 DB에 `npm run db:seed` 1회 필요.**
- **레이아웃 점검**: 부위당 16개면 chest/abdomen(3열)은 6행이 되어 인접 클러스터와 겹칠 수 있음
  → `atlas-layout.ts`에서 열 수(3→4) 또는 클러스터 간격 조정 후 스크린샷으로 확인.
