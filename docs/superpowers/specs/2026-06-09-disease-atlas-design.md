# Disease Atlas — 설계 문서

- **작성일:** 2026-06-09
- **상태:** 설계 확정 (구현 계획 작성 전)
- **참고:** https://lattice-log.vercel.app/atlas (그래프형 용어 지도)

## 1. 한 줄 정의

인체 부위별 주요 질병을 **해부학적으로 배치된 관계 그래프**로 한눈에 보고, 질병의
**용어·증상·치료법**을 검색·열람하며, 관리자가 웹에서 직접 콘텐츠를 채우는 교육용 사이트.

## 2. 목적과 성공 기준

- 사용자가 그래프만 봐도 "위=머리, 가운데=가슴" 식으로 **부위를 즉시 인지**할 수 있다.
- 질병 노드를 클릭하면 용어·증상·치료법·관련 질환을 한 패널에서 본다.
- `/` 검색으로 질병/증상을 빠르게 찾는다.
- 관리자가 코드 수정 없이 웹에서 질병을 추가·수정·삭제할 수 있다.

## 3. 핵심 화면

### Atlas (메인)
- 그래프 우선. 노드를 **신체 위치대로 고정 배치** (위=뇌·신경, 가운데=가슴, 아래=복부, 양옆=관절, 별도=내분비 등).
- 부위별 **색** + **클러스터 라벨** + 뒤에 흐릿한 **몸 실루엣 가이드**.
- 상단 **부위 필터칩**으로 특정 부위만 표시.
- 드래그로 이동, 스크롤로 줌, `/`로 검색.
- 노드 클릭 → 우측 상세 패널.

### 상세 패널
- 질병명, 부위, 계통/분류, **증상**, **치료법**, **관련 질환**(클릭 시 그래프에서 해당 노드로 이동).
- 상단에 **"의학적 조언 아님 · 교육용" 고지** 고정 노출.

### Admin (`/admin`)
- 단일 비밀번호 로그인(env).
- 질병·증상·계통·관계를 추가/수정/삭제하는 폼(CRUD).

## 4. 데이터 모델 (Prisma)

| 모델 | 주요 필드 | 설명 |
|------|-----------|------|
| `Disease` | name, description, treatment, bodyPartId, categoryId | 질병 본체 |
| `BodyPart` | name, color, layoutZone | 부위 + 그래프 좌표 영역/색 |
| `Category` | name | 계통/분류 (감염·자가면역·암 등) |
| `Symptom` | name | 증상 마스터 |
| `DiseaseSymptom` | diseaseId, symptomId | 질병↔증상 다대다 |
| `DiseaseRelation` | fromId, toId, type | 합병/연관 질환 명시적 링크 |

### 엣지(연결선) 생성 규칙
- **자동 파생:** 같은 부위 / 공통 증상 / 같은 계통
- **수동:** 합병·연관 질환 (`DiseaseRelation`)
- 엣지 **타입별 색/필터**로 구분 (4가지 관계 기준 모두 지원)

## 5. 기술 구성

- **프레임워크:** Next.js (React) 풀스택 — 화면 + API Route Handlers 한 코드베이스
- **ORM/DB:** Prisma + SQLite (개발) → Postgres (배포)
- **그래프:** React Flow — 고정 좌표 배치 + 커스텀 노드 + 줌/팬 + SVG 실루엣 배경 오버레이에 적합. *대안: Cytoscape.js*
- **검색:** Fuse.js 클라이언트 퍼지 검색, `/` 단축키
- **인증:** 관리자 단일 비밀번호(env). v1은 단순하게, 추후 확장
- **배포:** Vercel
- **언어:** 한국어 UI

## 6. v1 범위 (YAGNI)

**포함**
- 부위 5개 내외 + 시드 질병 20~30개
- Atlas 그래프 / 상세 패널 / 검색 / Admin CRUD
- 교육용·비의학적 고지

**제외 (나중에)**
- 다국어, 회원가입/소셜 로그인, 즐겨찾기, 댓글
- 학습 경로(Learning Path) — v2 후보

## 7. 안전장치

- 모든 질병·상세 화면에 **교육용·비의학적 고지** 고정 노출.
- 콘텐츠는 정확성 책임이 관리자에게 있음을 명시.

## 8. 결정 로그

- 데이터 관리: **웹 관리자 CRUD + DB** (정적 JSON·AI 자동생성 대신)
- 관계 기준: **같은 부위 + 공통 증상 + 합병/연관 + 같은 계통** (전부)
- 레이아웃: **그래프 우선(C) + 해부학적 고정 배치(1안)**
- 스택: **Next.js 풀스택**
